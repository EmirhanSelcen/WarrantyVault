# WarrantyVault

WarrantyVault is a full-stack web application for tracking personal products, warranty dates and service history.
Users register, login with JWT authentication and can only manage their own products and service records.

## Main Features

- Register and login with JWT
- User-specific data isolation
- Product CRUD operations
- Product-linked service records
- Search by product name, brand or serial number
- Category filter
- "Expiring in 30 days" warranty filter
- Vanilla JavaScript single-page frontend
- Express REST API with JSON responses
- SQLite database
- Swagger UI API documentation
- Unit tests for validation and warranty business logic

## Tech Stack

- Frontend: HTML, CSS, Vanilla JavaScript
- Backend: Node.js, Express
- Database: SQLite
- Authentication: JWT and bcryptjs
- API Docs: Swagger UI
- Tests: Jest

## Setup

1. Install Node.js from https://nodejs.org
2. Open the project folder in VS Code.
3. Install dependencies:

   ```bash
   npm install
   ```

4. Start the application:

   ```bash
   npm start
   ```

5. Open the app:

   ```text
   http://localhost:3000
   ```

6. Open Swagger API documentation:

   ```text
   http://localhost:3000/api-docs
   ```

## API Overview

Protected endpoints require this header:

```text
Authorization: Bearer <jwt-token>
```

### Auth

```http
POST /api/auth/register
POST /api/auth/login
```

### Products

```http
GET /api/products
GET /api/products/:id
POST /api/products
PUT /api/products/:id
DELETE /api/products/:id
```

Optional product query parameters:

```text
search=laptop
category=Electronics
expiringSoon=true
```

### Service Records

```http
GET /api/service-records
GET /api/service-records?productId=1
POST /api/service-records
DELETE /api/service-records/:id
```

## Project Structure

```text

src/config/          Environment and Swagger configuration
src/controllers/     Request/response layer
src/db/              SQLite initialization
src/middleware/      JWT authentication middleware
src/routes/          REST route definitions
src/services/        Business logic
src/utils/           Validation and warranty helpers
tests/               Jest unit tests
```
